TP — Premiers pas sur un serveur Debian

📘 Référentiel BTS SIO SISR
Compétences :
• B2.2 - Installer, tester et déployer une solution d'infrastructure
• B2.3 - Exploiter, dépanner et superviser une solution d'infrastructure
• B3.1 - Protéger les données (gestion des permissions)
🎯 Objectif

Préparer un serveur Debian à l'hébergement de l'application GLPI : arborescence de travail, fichiers, permissions, paquets et déploiement d'une archive.

⏱️ Durée

1 h 30 — notation automatisée sur 20 points.

🔧 Environnement

VM Debian 12 sans interface graphique, utilisateur etudiant, accès root. Connexion Internet requise.

⚙️ Préparation — Créer l'utilisateur etudiant

Les exercices se font avec un compte nommé etudiant. S'il n'existe pas encore sur la VM, créez-le en root :

adduser etudiant

La commande demande un mot de passe (à saisir deux fois), puis quelques informations facultatives (nom complet, etc.) : appuyez sur Entrée pour les laisser vides et confirmez par o. Elle crée aussi le dossier personnel /home/etudiant.

Connectez-vous ensuite en tant qu'etudiant :

su - etudiant

💡 Pour les exercices 4 à 6, vous repasserez root avec su - ; pour revenir à l'utilisateur etudiant, tapez exit.

🔑 Passer administrateur (root)

Certaines commandes (création de compte, permissions, installation, /var/www) exigent les droits d'administrateur. On devient root avec :

su -

Saisissez le mot de passe root (défini à l'installation de Debian). L'invite change alors de forme : elle se termine par $ pour un utilisateur normal, et par # pour root. Pour revenir à l'utilisateur etudiant, tapez exit.

🧭 Ce que l'on va faire

Vous allez transformer une VM Debian « nue » en un serveur prêt à héberger GLPI. Après avoir créé le compte etudiant (ci-dessus), vous enchaînerez 6 étapes, chacune correspondant à un exercice noté :

  1. Arborescence de travail — créer les dossiers et fichiers de base (mkdir, touch).
  2. Redirections — écrire dans des fichiers avec echo et les redirections > / >>.
  3. Copier / déplacer — organiser les fichiers (cp, mv).
  4. Permissions — régler les droits en notation octale et changer le groupe propriétaire (chmod, chgrp).
  5. Paquets — mettre à jour et installer des logiciels (apt).
  6. Déploiement de GLPI — télécharger et extraire l'application dans /var/www (wget, tar, chown).

Enfin, vous lancerez un script de correction automatique qui vérifie l'état de votre VM et produit le fichier à déposer sur Moodle.

⚠️ Règles du TP

Tout se fait en ligne de commande. Les noms de fichiers, dossiers et permissions doivent être strictement identiques à l'énoncé : le script de correction vérifie l'état exact de votre VM. Respectez les majuscules/minuscules. Tapez vous-même les commandes plutôt que de les copier-coller : c'est ainsi que l'on apprend.

Exercice 1 — Arborescence de travail (3 pts)

Connecté en tant qu'utilisateur etudiant :

  1. Dans votre répertoire personnel, créez un dossier tp-linux.
  2. Dans tp-linux, créez trois sous-dossiers : docs, scripts et archives.
  3. Dans docs, créez un fichier vide nommé notes.txt.

Commandes :

cd ~ mkdir tp-linux cd tp-linux mkdir docs scripts archives touch docs/notes.txt

💡 ~ désigne votre dossier personnel (/home/etudiant). mkdir accepte plusieurs dossiers d'un coup. touch crée un fichier vide.

Exercice 2 — Redirections (3 pts)

  1. Dans tp-linux, créez un fichier identite.txt contenant votre nom et prénom, en une seule commande avec echo et une redirection.
  2. Ajoutez (sans écraser !) une seconde ligne contenant exactement : BTS SIO SISR

Commandes (remplacez Nom Prénom par le vôtre) :

cd ~/tp-linux echo "Nom Prénom" > identite.txt echo "BTS SIO SISR" >> identite.txt

💡 > écrase le fichier (le crée s'il n'existe pas), >> ajoute à la fin. Vérifiez le résultat avec cat identite.txt.

Exercice 3 — Copier, déplacer (3 pts)

  1. Copiez identite.txt dans docs/ sous le nom identite-copie.txt.
  2. Déplacez docs/notes.txt dans archives/ (il ne doit plus exister dans docs).

Commandes :

cd ~/tp-linux cp identite.txt docs/identite-copie.txt mv docs/notes.txt archives/

💡 cp source destination copie, mv source destination déplace (ou renomme). Vérifiez avec ls docs archives.

📝 Éditer un fichier avec nano

À partir d'ici, vous aurez parfois besoin d'éditer un fichier. nano est un éditeur de texte en console. Pour ouvrir un fichier (ou le créer s'il n'existe pas encore) — ici fichier.txt n'est qu'un nom d'exemple, à remplacer par le fichier voulu :

nano fichier.txt

On se déplace avec les flèches. Les raccourcis utilisent la touche Ctrl (notée ^ en bas de l'écran) :

  • Ctrl+O puis Entrée — enregistrer
  • Ctrl+X — quitter
  • Ctrl+K — couper la ligne · Ctrl+U — la coller
  • Ctrl+W — rechercher

Exercice 4 — Permissions (4 pts)

Les changements de groupe nécessitent des privilèges : passez root le temps de cet exercice.

  1. Donnez à tp-linux/identite.txt : lecture/écriture pour le propriétaire, lecture seule pour le groupe, aucun droit pour les autres.
  2. Donnez au dossier tp-linux/scripts : tous les droits pour le propriétaire, lecture + traversée pour le groupe, aucun droit pour les autres.
  3. Changez le groupe propriétaire de docs/identite-copie.txt pour www-data.

Rappel octal — chaque chiffre = somme de r(4) + w(2) + x(1) :

rw- r-- --- = 6 4 0 → 640 (identite.txt) rwx r-x --- = 7 5 0 → 750 (dossier scripts)

Commandes (en root) :

su - cd /home/etudiant/tp-linux chmod 640 identite.txt chmod 750 scripts chgrp www-data docs/identite-copie.txt

💡 Vérifiez avec ls -l (fichiers) et ls -ld scripts (dossier). Restez en root pour les exercices 5 et 6.

Exercice 5 — Gestion des paquets (2 pts)

  1. Mettez à jour la liste des paquets disponibles.
  2. Installez les paquets wget et tree.
  3. Vérifiez avec tree que votre arborescence correspond à celle des exercices 1 à 3.

Commandes (en root) :

apt update apt install -y wget tree tree /home/etudiant/tp-linux

💡 apt update rafraîchit le catalogue, apt install installe. L'option -y répond « oui » automatiquement.

Exercice 6 — Déploiement de GLPI (5 pts)

Toujours en root :

  1. Téléchargez l'archive GLPI directement dans /var/www.
  2. Extrayez-la de façon à obtenir le dossier /var/www/glpi.
  3. Donnez récursivement /var/www/glpi à l'utilisateur et au groupe www-data.

Commandes :

cd /var/www wget https://github.com/glpi-project/glpi/releases/download/10.0.6/glpi-10.0.6.tgz tar -xvf glpi-10.0.6.tgz chown -R www-data:www-data /var/www/glpi

💡 En se plaçant dans /var/www avant l'extraction, tar y crée le dossier glpi/. Variantes équivalentes : wget -P /var/www <url> pour choisir la destination, et tar -xvf archive.tgz -C /var/www pour extraire ailleurs que dans le dossier courant. chown -R utilisateur:groupe applique récursivement le propriétaire et le groupe.

✅ Validation

Quand tout est terminé, lancez le script de correction en root :

python3 check_tp_linux.py

Le script affiche votre détail par exercice et génère un fichier resultat_tp.json à déposer sur Moodle. Vous pouvez le relancer autant de fois que nécessaire avant le dépôt.

🔌 Pour aller plus loin — se connecter en SSH (PuTTY)

Maintenant que vous avez réalisé le TP en tapant les commandes directement dans la console de la VM, voici comment vous y connecter à distance depuis le PC hôte avec PuTTY — pratique pour les prochains TP. Avec VMware Workstation en NAT, l'hôte joint directement l'IP de la VM, sans configuration réseau supplémentaire.

1. Installer le serveur SSH sur la VM (en root) :

su - apt update apt install -y openssh-server

2. Relever l'adresse IP de la VM :

ip a

Cherchez l'adresse inet de l'interface réseau (souvent ens33) — sous VMware Workstation NAT, elle ressemble à 192.168.x.x.

3. Se connecter avec PuTTY (sur l'hôte) :

  • Host Name : l'IP relevée (ex. 192.168.239.128)
  • Port : 22
  • Connection type : SSH → cliquez sur Open

Connectez-vous avec le compte etudiant et son mot de passe (par défaut, Debian n'autorise pas la connexion SSH directe en root : connectez-vous en etudiant puis faites su -).

4. Mettre le système à jour (bonne pratique après connexion, en root) :

su - apt update apt upgrade -y

💡 apt update rafraîchit le catalogue des paquets ; apt upgrade installe les mises à jour disponibles. -y valide automatiquement.
